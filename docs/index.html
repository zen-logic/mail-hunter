Why Mail Hunter

One archive, every account

Connect live IMAP servers for incremental sync. Import EML files and MBOX exports for everything else. Twenty years of email across a dozen providers, searchable from a single browser tab.

Your mail, your disk

Every message is stored locally in a content-addressable archive compressed with zstd at level 19 — typically 40–60% of the original size. Deduplicated by SHA-256 hash so the same message across five servers is stored once. No cloud dependency, no subscription, no provider lock-in.

Search everything at once

Full-text search across all servers, all folders, all years. Filter by sender, recipient, subject, body, date range, attachment filename, tags, duplicate status, or legal hold. Save searches for one-click re-use. Results come back fast because the index is local SQLite, not a remote API with rate limits.

Features

IMAP Sync

Incremental sync via UID checkpoints. Full resync and purge options. Configurable auto-sync intervals per server. Sync queue ensures one operation at a time. Progress streamed live over WebSocket. Cancel and resume at any time.

Import

Drag and drop EML files or MBOX archives anywhere in the app. Automatic address matching to existing servers. Batch import with progress tracking and duplicate detection.

Deduplication

Messages hashed by Message-ID and content. Identical messages across servers and folders stored once on disk. Duplicate badges visible at a glance — click to see every copy with its server and folder location.

Conversation Threading

Select any reply and click “Show Thread” to see the full conversation reconstructed from In-Reply-To and References headers, spanning servers and folders.

Compression

Raw EML files compressed with zstd at level 19. Content-addressable storage keyed by SHA-256. Legacy uncompressed archives continue to work without migration.

Tagging

Tag individual messages or batch-tag entire selections. Organise across server and folder boundaries. Gmail labels imported automatically via X-GM-LABELS backfill.

Legal Hold

Place individual messages on legal hold to prevent deletion. Held messages survive batch deletes, folder purges, and server removal. For compliance, disputes, or anything you can’t afford to lose.

Message Preview

View HTML (with inline CID images), plain text, and raw RFC822 source in a full-screen sandboxed preview. Download the original EML or extract individual attachments.

Real-Time UI

Three-panel layout with resizable panes. Sync progress, folder counts, and system status update live via WebSocket. Keyboard navigation and multiple colour themes.

Batch Operations

Select multiple messages with click, shift-click, or select-all. Batch delete, batch tag, batch hold, and batch export as zip. Export archive folders as standard MBOX files. Legal hold respected across all bulk operations.

Dashboard

Global stats with per-server breakdown of message counts, duplicates, held messages, and storage usage. Click any server to navigate directly.

Encrypted Credentials

IMAP passwords encrypted at rest with Fernet symmetric encryption. Key generated on first run, stored in your config.
